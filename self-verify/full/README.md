# OntoDerive 全量自举验证

> 验证范围：OntoDerive框架的全部29个源文件
> 验证目标：框架文档描述的能力、引擎实际执行的能力、元模型定义的类型三者之间一致
> 核心路径：文档 → 事实/实体/推论 → 引擎验证 → 结果与文档对比

---

## 自举循环

```
[01] 框架文档(README/CLAUDE/元模型/方案)
    ↓ agent读取并提取
[02] 事实基座 + 实体 + 推论
    ↓ derive.py 执行
[03] 规约检查结果(8条C-xx)
    ↓ 与文档声明对比
[04] 一致性报告 → 文档是否需要更新？
    ↑ 循环
```

## 验证维度

| 维度 | 说明 | 数据来源 |
|------|------|---------|
| 文件完整性 | 框架所有文件是否存在 | find / framework |
| 元类型覆盖 | 10元类型是否在文档中全部声明 | 元模型定义.md |
| 功能声明与实现 | README声称的能力引擎是否实现 | README.md → derive.py |
| 断言一致性 | 文档断言是否有编号引用 | 所有.md文件 |
| 示例可运行 | examples/z-park能否通过验证 | 示例目录 |
| 自举循环 | 自验证能否发现自身的缺口 | 本目录 |
