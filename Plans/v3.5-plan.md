# OntoDerive v3.4 → v3.5 产品化与深化方案

> 基于全量代码审查 + 文档差距分析 + 测试/质量扫描
> 研究日期: 2026-05-20
> 当前状态: v3.4.0, 197 tests pass, 8,464行, 180 ruff issues

---

## 研究摘要

### 已完成 (v3.3→v3.4)
P1-P4全部执行完毕: R19循环修复 + derivation_trail + AnalyticsEngine统一接入 + SemanticMatcher + RuleLoader YAML + semantic_depth连续推理 + 死代码清理。

### 核心发现
1. **代码质量漂移**: 180条ruff错误积压，多数是F401(未使用导入)和F841(未使用变量)
2. **SemanticMatcher重复**: `engine/intelligence/semantic.py`与`engine/foundation/semantic.py`完全一致
3. **版本不一致**: MCP server报3.3.0，engine/__init__.py文档字符串说v3.3，实际v3.4.0
4. **规则硬编码**: 19条规则中仅4条(R22-R25)YAML化，其余全在Python中
5. **分析模式少**: 仅A1-A5五个模式，且全部硬编码
6. **产出格式单一**: 仅Markdown报告，无HTML/JSON结构化输出
7. **CHANGELOG滞后**: 仅覆盖到v3.0

## 执行计划

### P0: 代码质量收敛 (1.5h)

**0a. 版本号统一** (~10行)
- `engine/__init__.py`: 文档字符串 v3.3 → v3.5, `__version__` = "3.5.0"
- `engine/core/derive.py`: VERSION = "3.5.0"
- `engine/mcp_server.py`: serverInfo.version "3.3.0" → "3.5.0"
- `engine/cli.py`: 描述文本 v3.4.0 → v3.5.0
- `pyproject.toml`: version "3.3.0" → "3.5.0"

**0b. SemanticMatcher去重** (~5行)
- 统一为 `engine/intelligence/semantic.py`
- `engine/foundation/semantic.py` → 改为 re-export: `from engine.intelligence.semantic import SemanticMatcher`
- `engine/theories/analytics.py`: 确认导入路径一致

**0c. Ruff清理** (~100行改动)
- F401: `engine/__init__.py` 显式re-export (`as OntoDerive`等)
- F841: 未使用变量清理
- F811: 重复定义修复
- 目标: 180 → <20 errors (仅保留有意的)

### P1: 规则YAML化扩展 (3h)

**1a. 规则迁移 (~200行YAML)**
- 从 `RuleReasoner` 中提取R1-R19规则定义, 写为YAML
- `engine/foundation/rules/core_rules.yaml` — 确定性规则(R1-R9)
- `engine/foundation/rules/heuristic_rules.yaml` — 启发式规则(R10-R14)
- `engine/foundation/rules/structural_rules.yaml` — 结构性规则(R15-R19)
- 每个YAML规则含: id/name/type/category/premises/conclusion_template/confidence

**1b. RuleReasoner接入RuleLoader** (~50行)
- `RuleReasoner.__init__` 接受 `rule_loader` 参数
- 加载的YAML规则以 `to_conclusion()` 转换为推理结果
- 保持向后兼容: 无YAML时使用内置硬编码规则

**1c. 新增规则 (~5条)**
- R26: 区间算术 (Interval Arithmetic)
- R27: 变化率检测 (Rate of Change)
- R28: 基准偏离 (Benchmark Deviation)
- R29: 集中度风险 (Concentration Risk) — 从analytics_rules.yaml提升
- R30: 缺失值检测 (Missing Value Detection)

### P2: 分析模式深化 (4h)

**2a. YAML化现有模式** (~100行YAML)
- `engine/foundation/rules/analytics_rules.yaml` 扩展现有R22-R25
- A1-A5模式定义写入YAML: detect条件 + analyze逻辑参数化

**2b. 新增分析模式 A6-A8** (~250行)
- **A6: 市场结构分析** (MarketStructure)
  - 检测: 实体数量、份额分布、进入壁垒指标
  - 输出: HHI集中度、市场类型判定、壁垒评分
  - semantic_depth: 0 (纯计算)
- **A7: 博弈均衡检测** (GameEquilibrium)
  - 检测: 多方互动关系(竞争/合作/博弈)
  - 输出: 纳什均衡可能性、囚徒困境识别、协调博弈判定
  - semantic_depth: 2 (TF-IDF + 轻量分类)
- **A8: 策略选项生成** (StrategicOptions)
  - 检测: 目标+约束+资源组合
  - 输出: 可行策略列表 + 风险/收益评分 + 敏感性分析
  - semantic_depth: 3 (轻量分类器 + optional LLM)

**2c. AnalyticsEngine YAML注册** (~80行)
- 支持从YAML加载分析模式定义
- `detect`和`analyze`的Python函数通过注册表映射

### P3: 产品化打磨 (3h)

**3a. 多格式输出** (~150行)
- `engine/core/export.py` — 统一导出模块
- `--export json`: JSON结构化输出 (facts + inferences + checks)
- `--export html`: 单文件HTML报告 (内联CSS, 可浏览器查看)
- CLI: `ontoderive generate --project . --export html`

**3b. 提取器增强** (~100行)
- `engine/extractor.py` 已存在 — 增强:
  - URL下载 (urllib, 无外部依赖)
  - 基础HTML→文本转换 (regex based, 零依赖)
  - 自动分块 (按标题/段落)

**3c. CHANGELOG补全** (~60行)
- v3.1: RuleReasoner 19规则 + FormalReasoner
- v3.2: 四阶段管线 + OntoLang解析器 + 形式推理
- v3.3: 五层物理拆分 + LLM主提取引擎 + 语义域过滤器 + 本体建模
- v3.4: P1-P4推理体系升级 (循环修复+可解释性+三引擎统一+语义匹配+规则YAML化)

**3d. CLI体验提升** (~50行)
- 彩色输出 (ANSI颜色码, 零依赖)
- 进度提示 (spinner for long operations)
- `--quiet` / `--verbose` 标志

### P4: 高级推理能力 (4h)

**4a. 加权影响传播** (~120行)
- `engine/theories/graph_reasoning.py` — 新建图推理模块
- 基于derives_from DAG做加权影响传播
- 事实变更→自动计算影响的推论置信度变化
- 衰减系数: 每跳 ×0.9

**4b. 时态推理增强** (~100行)
- `engine/reasoners/reasoner.py` R15扩展
- Allen区间关系的before/after/meets/overlaps
- 检测"新事实推翻旧推论"模式
- 检测"推论基于已废止政策"

**4c. 证据合成** (~80行)
- `engine/theories/evidence.py` — 新建证据合成模块
- 多源置信度聚合 (加权平均 + Dempster规则简化版)
- 冲突检测: 两源置信度差异 >0.4 标记为冲突
