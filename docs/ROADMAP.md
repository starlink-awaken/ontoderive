## 二、下一阶段规划：v2.1 "智能层"

### Phase 0: 基础巩固 (1周)

| 任务 | 优先级 | 说明 |
|------|--------|------|
| 修复C-07 INFERENCE误报 | P0 | 正则改进, INFERENCE vs INF-编号 |
| 完善docs自验证至8/8 | P0 | 补齐断言追溯至100% |
| 规范Issue/PR模板 | P1 | CONTRIBUTING.md更新 |

### Phase 1: 贝叶斯层 (3天)

**目标**: 置信度从high/medium/low → 连续概率 P(e) ∈ [0,1]

```python
# 输入: 当前置信度标签
confidence: "high"  # 固定值

# 输出: 概率分布
confidence: 0.92    # 连续概率
```

具体任务:
1. 设计概率本体: 每个实体P(e) + 每个关系P(e2|e1)
2. 信念传播: 当D-Fx更新时, 自动更新依赖INF的置信度
3. 引擎集成: check输出包含置信度分布

**交付物**:
- `framework/03-推理引擎/02-贝叶斯层.md` — 设计文档
- `engine/bayesian.py` — 信念传播模块
- 验证: 在self-verify/docs上运行, 置信度应自动从事实传播到推论

### Phase 2: 信息论层 (2天)

**目标**: 知识质量指数(KQI), 熵计算, 信息增益引导

```python
# 输入: 知识库当前状态
KQI = {
    "entropy": 3.45,        # H(KB) 知识库总熵
    "precision": 0.82,      # 平均置信度
    "density": 0.45,        # 推导深度(推论/事实比)
    "coverage": 0.87,       # 映射覆盖率
    "closure_rate": 0.75,   # 缺口闭环率
}
```

**交付物**:
- `framework/03-推理引擎/03-信息论层.md`
- `engine/metrics.py` — 熵计算+KQI指数
- 验证: z-park的KQI应该比docs低(项目复杂度不同)

### Phase 3: 控制论层 (5天)

**目标**: PID反馈控制 + 自适应阈值 + 自演化学习

| 分量 | 含义 | 实现 |
|------|------|------|
| P(比例) | 当前违规数 → 立即修复 | 已实现(resolve) |
| I(积分) | 历史违规累积 → 系统性问题 | 新增: 规约自调整 |
| D(微分) | 违规变化率 → 预测性修复 | 新增: 趋势告警 |

**交付物**:
- `framework/03-推理引擎/04-控制论层.md`
- `engine/controller.py` — PID控制器
- `engine/learner.py` — 自演化(规约自适应阈值)
- 验证: 运行--rounds 5, 收敛速度应比v1.2快

### Phase 4: 图灵机层 (4天)

**目标**: 知识状态机, 快照管理, 可重放

```python
K-TM = {
    "snapshots": [],        # q0, q1, q2, ... 知识状态历史
    "transitions": [],      # δ0, δ1, δ2, ... 推导步骤
    "halt_reason": "converged",  # 停机原因
}
```

**交付物**:
- `framework/03-推理引擎/05-图灵机层.md`
- `engine/turing_k.py` — 知识状态机
- 验证: 4轮推导后快照差异应逐轮缩小

### Phase 5: 形式语言层 (5天)

**目标**: OntoLang解析器, BNF语法检查

```python
# 输入: OntoLang DSL
entity ORG-国转中心 : Organization { governance: "双轨制" }
fact D-F1 : DataPoint { value: 470 }

# 输出: 解析树 + 语法错误
```

**交付物**:
- `framework/02-形式语言/01-OntoLang-BNF.md`
- `engine/ontolang/` — 基于lark的解析器
- 验证: 100个测试用例错误率<5%

## 三、优先级建议

| 优先级 | Phase | 理由 | 依赖 |
|--------|-------|------|------|
| 🔴 P0 | 基础巩固 | 当前已知问题的热修复 | 无 |
| 🔴 P1 | 贝叶斯层 | 置信度是其他层的基础输入 | P0 |
| 🟡 P2 | 信息论层 | KQI是衡量其他层效果的标准 | P1 |
| 🟡 P3 | 控制论层 | 需要贝叶斯层的置信度作为输入 | P1+P2 |
| 🟢 P4 | 图灵机层 | 需要控制论的稳定性分析 | P3 |
| 🟢 P5 | 形式语言层 | 独立性最强,可并行 | 无 |

**推荐路线**: P0→P1→P2→P3→P4→P5 (约19-24天)
