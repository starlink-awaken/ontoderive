# OntoDerive 产品化分析——面向目标的持续推导引擎

> 场景：输入目标+上下文+约束 → 持续迭代 → 输出事实+方法+推论+结果
> 变化驱动：事实修正/方法迭代/策略变更/约束变更 → 自动重新生成

---

## 一、场景分析

### 用户工作流

```
[目标] + [上下文目录/链接] + [策略/约束]
    ↓
信息提取（facts/ 自动生成）
    ↓
方法匹配（从方法论库自动选择或用户指定）
    ↓
推导迭代（多轮 --rounds N）
    ↓
结果产出（scheme/ + KQI评估）
    ↓
变更触发（事实/方法/策略/约束任一变化）
    ↓
重新生成
```

### 核心需求矩阵

| 需求 | 当前支撑 | 缺口 |
|------|---------|------|
| 目标输入 | ⚠️ --init 只创建空骨架 | 需要 --goal 参数驱动初始化 |
| 上下文提取 | ❌ 无自动提取 | 需要本地文件/网页解析 |
| 策略匹配 | ❌ 无策略库 | 需要方法论匹配引擎 |
| 自动推导 | ✅ --derive + --rounds | 能力完备 |
| 产出交付 | ⚠️ --generate report | 需要多形态产出（MD+PDF+DOCX） |
| 变更重生成 | ❌ 无变化检测 | 需要文件监听+自动触发 |
| 事实修正 | ⚠️ 手动编辑 facts/ | 需要结构化UI/CLI |
| 方法迭代 | ⚠️ 手动更新 inferences/ | 需要版本化方法论库 |

## 二、现有工具支撑评估

| 能力 | 支撑度 | 说明 |
|------|--------|------|
| 项目初始化与骨架 | ✅ 80% | --init 创建6个文件，缺内容自动填充 |
| 事实基座管理 | ✅ 60% | 手动编辑，缺自动提取和格式校验 |
| 推论体系建立 | ✅ 70% | derives_from链 + C-04追溯检查 |
| 方案产出 | ✅ 60% | markdown报告，缺多格式和多形态 |
| 规约检查 | ✅ 90% | 13条规约覆盖完整 |
| 多轮迭代 | ✅ 80% | --rounds N 支持，缺自动收敛判断 |
| KQI质量评估 | ✅ 70% | C-10 KQI指数，缺历史对比 |
| 变化检测 | ❌ 10% | 无文件监听，无自动重跑 |
| 上下文自动提取 | ❌ 5% | 无网页解析，无文档解析 |
| 方法论自动匹配 | ❌ 5% | 无方法论库 |

## 三、产品架构设计

### 新增模块

```
engine/
├── derive.py          # 核心引擎 [现有]
├── bayesian.py        # 贝叶斯层 [现有]
├── metrics.py         # 信息论层 [现有]
├── controller.py      # 控制论层 [现有]
├── turing_k.py        # 图灵机层 [现有]
├── ontolang.py        # 形式语言层 [现有]
├── mcp-server.py      # MCP服务 [现有]
├── extractor.py       # [新增] 上下文提取器
├── matcher.py         # [新增] 方法论匹配器
└── watcher.py         # [新增] 文件监听器
```

### 新增命令

```bash
# 目标驱动初始化
ontoderive --goal "分析XXX市场" --context ./research/ --method "波特五力"

# 上下文自动提取
ontoderive --extract https://example.com/report --to facts/

# 增量变更
ontoderive --watch --auto-rerun  # 监听文件变化自动重推导

# 多形态产出
ontoderive --generate all --formats md,docx,pdf

# 变更对比
ontoderive --delta compare q10 q11  # 比较两次知识快照
```

### 用户旅程

```
1. 输入目标 → --goal 自动初始化项目
2. 提供上下文 → --extract 自动提取为 facts/
3. 确认方法 → --method 或自动匹配
4. 执行推导 → --rounds 5 多轮迭代
5. 审阅产出 → --generate all
6. 提出修正 → 手动编辑 facts/ 或 --extract 追加
7. 自动重跑 → --watch 或手动 --derive --check
8. 对比变化 → --delta 查看新版本变化
```

## 四、实施优先级

| 优先级 | 模块 | 价值 | 工作量 |
|--------|------|------|--------|
| P0 | extractor.py（上下文提取） | 最大的体验提升 | 3天 |
| P1 | --watch + 自动重跑 | 闭环的关键 | 2天 |
| P2 | --goal 驱动初始化 | 降低使用门槛 | 1天 |
| P3 | --delta 变更对比 | 让迭代可见 | 1天 |
| P4 | matcher.py（方法论匹配） | 策略自动推荐 | 3天 |
