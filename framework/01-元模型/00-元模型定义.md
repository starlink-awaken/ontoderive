# 元模型——OntoDerive的建模语言

> 元模型定义"建模的语言本身"——有哪些元类型、元关系、元约束。
> 它本身不包含任何具体实例，是对实例层的抽象。

---

## 一、八层架构

```
L7: 元元层(Meta-Meta) — "建模的语言本身"
L6: 元层(Meta) — "什么是模型"  
L5: 处理器层 — 规则和引擎
L4: 知识组织层 — 怎么组织和表达(文档/索引)
L3: 行为层 — 系统怎么运转(状态机/飞轮)
L2: 关系层 — 实体间的连接
L1: 实体层 — 具体实体实例
L0: 事实基座 — 原始数据与政策事实
```

## 二、10类元实体类型（v2扩展）

| 元类型 | 本质特征 | 包含实体 | 可做 | 不可做 |
|--------|---------|---------|------|-------|
| DOMAIN | 现实世界中的"东西" | ORG, ROL, PRJ, RES | part_of, participates_in | derives_from |
| FACT | 可验证的客观陈述 | DAT, POL | 被cites, 被derives_from | 主动推理 |
| INFERENCE | 基于事实的推导 | INF, IP | derives_from, maps_to | 直接cites事实 |
| RELATION | 连接实体的边 | 无独立实体 | 无 | 独立存在 |
| STATE | 系统行为节点 | T, F, H | transitions_to, interlocks_with | contains文档 |
| DOCUMENT | 知识组织形态 | DOC, CH, SEC | contains, maps_to | transitions_to |
| CONSTRAINT | 质量门禁 | CON, 规则 | 被satisfies | 有状态 |
| PROCESSOR | 操作模型的模型 | 推导引擎 | processes, generates | 有状态 |
| **PROBABILISTIC** | **概率分布** | **BAY-*, PRIOR-*** | **belief_update, entropy** | **确定性断言** |
| **METRIC** | **可量化指标** | **KQI-*, MEAS-*** | **measure, compare** | **生成推论** |

> v2扩展说明：自举验证发现PROBABILISTIC和METRIC两种新元类型是必需的。PROBABILISTIC承载贝叶斯网络和信念传播，METRIC承载信息熵和知识质量指数。

## 三、4类元关系

| 类别 | 语义 | 包含关系 | 特征 |
|------|------|---------|------|
| STRUCT | "由什么组成" | part_of, contains | 树形,有向,不可成环 |
| DERIVE | "从什么产出" | derives_from, maps_to | 可追溯,证据链 |
| BEHAVIOR | "什么触发什么" | transitions_to, interlocks_with | 有向,可循环 |
| JUSTIFY | "什么支撑什么" | cites, satisfies, depends_on | 非对称,约束性 |

## 四、4条元约束

| 约束 | 定义 | 违反后果 |
|------|------|---------|
| META-CON-1 类型纯度 | 每个实体属于且只属于一个元类型 | 类型污染→关系混乱 |
| META-CON-2 关系方向 | 关系必须遵守元关系矩阵 | 语义错误 |
| META-CON-3 处理器输入 | PROCESSOR输入必须是已实例化实体 | 空输入→错误产出 |
| META-CON-4 自指边界 | PROCESSOR不可处理自身代码 | 无限递归 |

## 五、元关系矩阵

```
被→  DOMAIN  FACT   INF     STATE  DOC   CONSTR  PROC
DOMAIN  S     D      D      —      —     —       —
FACT    —     —      —      —      J     —       —
INF     —     —      —      —      —     —       —
STATE   —     —      —      B      —     —       —
DOC     —     J      M      —      S     SA      —
CONSTR  —     —      —      —      —     —       —
PROC    —     —      G      —      G     V       —
```
S=STRUCT, D=DERIVE, J=JUSTIFY, B=BEHAVIOR, M=MAPS_TO, SA=SATISFIES, G=GENERATES, V=VALIDATES

## 六、扩展机制

元类型遵循**开放-封闭原则**：
- **封闭**：核心8类元类型不修改
- **开放**：可通过`MET-EXT-{name}`扩展自定义元类型，但必须遵守元关系矩阵
