# 中关村科学城服务平台方案 — OntoDerive 完整示例

> 本示例展示从零开始使用 OntoDerive 的完整流程。
> 场景：某科技园区需要建立一个科技成果服务平台。

---

## 四阶段推导全流程

### 阶段一：事实基座

**facts/data.md**
```markdown
| 编号 | 数据 | 数值 | 来源 |
|------|------|------|------|
| D-F1 | 园区入驻企业 | 240家 | 园区管委会 |
| D-F2 | 高校合作 | 12所 | 园区统计 |
| D-F3 | 年技术对接量 | 80次 | 运营报告 |
| D-F4 | 转化成功率 | 8% | 运营报告 |
```

**facts/policy.md**
```markdown
| 编号 | 政策 | 发布主体 | 日期 |
|------|------|---------|------|
| P-F1 | 科技成果转化法修正案 | 全国人大 | 2025 |
| P-F2 | 中关村先行先试政策 | 北京市 | 2026 |
```

### 阶段二：实体本体

**entities/actors.md**
```markdown
| 实体 | 类型 | 角色 | 数量 |
|------|------|------|------|
| ORG-园区管委会 | 组织 | 运营主体 | 1 |
| ORG-入驻企业 | 组织 | 需求方 | 240家(D-F1) |
| ORG-合作高校 | 组织 | 供给方 | 12所(D-F2) |
| ROL-技术经理人 | 角色 | 撮合人 | 20人 |
```

### 阶段三：推论体系

**inferences/01-contradictions.md**
```markdown
## L1 供需匹配矛盾

推导过程：
1. 240家企业有技术需求(D-F1)，12所高校有成果(D-F2)
2. 但年对接仅80次(D-F3)，转化率8%(D-F4)
3. 推论：缺乏系统化的供需匹配机制
4. 理论支撑：信息不对称理论(Akerlof, 1970)
5. 结论：需要数字化对接平台
- derives_from: [D-F1, D-F2, D-F3, D-F4]
```

**inferences/02-derivations.md**
```markdown
## 推导一：从L1到对接平台设计

- 矛盾来源：L1供需匹配矛盾
- 理论支撑：平台商业模式(Parker, 2016)
- 设计结论：建设在线对接平台，提供智能匹配+流程管理
- maps_to: scheme/01-平台方案.md
```

### 阶段四：方案产出

**scheme/01-平台方案.md**
```markdown
# 科技成果对接平台建设方案

## 建设目标
建设一个覆盖成果发布、智能匹配、对接跟踪的数字化平台(D-F1~D-F4)。

## 核心功能
1. 成果库：高校可发布技术成果(ROL-合作高校)
2. 需求库：企业可发布技术需求(ROL-入驻企业)
3. 智能匹配：基于语义技术实现自动推荐(INF-L1)
4. 对接管理：全程跟踪撮合进度(ROL-技术经理人)
```

### 执行引擎

```bash
cd examples/z-park
python3 ../../engine/derive.py --derive     # 正向推导
python3 ../../engine/derive.py --check      # 规约检查
python3 ../../engine/derive.py --generate report  # 生成报告
```

## 关键设计原则

1. **编号体系**：D-Fx(数据事实)/P-Fx(政策事实)/ORG-x(组织)/ROL-x(角色)等
2. **推导链**：每个推论标注 derives_from 事实编号
3. **追溯性**：方案中每个断言引用来源编号
4. **可证伪**：核心假设标注"如果X则重新评估"条件

## 应用到新领域

| 步骤 | 动作 | 产出 |
|------|------|------|
| 1 | 收集原始数据 | facts/data.md + facts/policy.md |
| 2 | 识别关键实体 | entities/ 下的md文件 |
| 3 | 分析矛盾/问题 | inferences/contradictions.md |
| 4 | 提出解决方案 | inferences/derivations.md |
| 5 | 编写方案 | scheme/ 下的md文件 |
| 6 | 运行引擎 | `derive.py --derive --check` |
| 7 | 迭代收敛 | `derive.py --rounds 3` |
